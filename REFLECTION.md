# Reflection on Using AI Agents

## What I Learned Using AI Agents
Using AI agents drastically shifts the development paradigm from manually typing every line of code to focusing on system design, prompt engineering, architecture modeling, and code review. I learned how to effectively break down complex business logic—such as the rules governing FuelEU Maritime regulations across banking, borrowing, and pooling—into well-defined chunks and clear instructions rather than writing the imperative logic manually. Directing an AI requires high-level oversight; I discovered that being explicit about design goals and project structures (like Hexagonal Architecture) ensures that the generated codebase remains highly cohesive and properly decoupled, preventing it from devolving into messy code.

## Efficiency Gains vs Manual Coding
The efficiency gains were substantial across the entire stack. Bootstrapping the application, scaffolding the hexagonal layers (ports and adapters), writing the Prisma schemas, and implementing repetitive UI components with Tailwind CSS were accomplished in a fraction of the standard time. Tasks that would normally take hours or days were completed rapidly, significantly accelerating the timeframe from concept to working prototype. However, I noticed that complex edge cases, specific domain-specific constraints, and minute bug-hunting still required active manual guidance, code review, and oversight. The AI serves as an incredibly powerful accelerator and pair programmer, but not an entirely autonomous replacement for developer logic.

## Improvements I'd Make Next Time
Next time, I would focus more extensively on Test-Driven Development (TDD) as the primary way of steering the AI. By prompting the AI to write the test suites (especially boundary and unit tests for the core domain) *first*, I can ensure its subsequent implementations are strictly bound to the expected input and output criteria, reducing the potential for hallucinated logic or regressions. Additionally, I would adopt a more incremental approach; providing smaller, more constrained scopes per prompt to the agent inherently reduces instances where the AI tackles too much at once and loses context regarding the wider architectural boundaries.
